# Heart Disease Prediction with Neural Networks

## The problem

Heart disease is the leading cause of death worldwide. Early identification significantly increases the chances of successful treatment, but traditional diagnosis depends on multiple exams and the availability of specialists.

This project trains a machine learning model capable of estimating the probability of a patient having heart disease based on 13 routine clinical variables (age, blood pressure, cholesterol, electrocardiogram, and others). The result is delivered in seconds, via web interface or API.

## What the model delivers

- **Probability of heart disease** for a specific patient (e.g. 78%)
- **Binary diagnosis**: disease or no disease, based on an adjustable threshold
- **Prediction explanation**: which clinical variables contributed most to that result, patient by patient

## Results

The model was evaluated on data it never saw during training (test set):

| What it measures | Result | What it means |
|---|---|---|
| AUC-ROC | ~0.92 | In 92% of cases, the model correctly distinguishes diseased from healthy patients |
| Accuracy | ~0.85 | 85% of predictions are correct |
| Recall (disease) | ~0.87 | The model identifies 87% of patients with disease, minimizing undetected cases |

The focus on **recall** is intentional: in a clinical context, missing a sick patient is far more costly than a false alarm.

## How it works in practice

The doctor or operator fills in the patient's clinical data on the web interface. In under a second, the system returns the probability of disease and a visual explanation showing which factors weighed most in that diagnosis, for example, "elevated ST depression and the number of affected vessels were the main risk indicators".

This does not replace professional medical diagnosis, but works as a triage and decision support tool.

## Setup

All commands must be run from the **project root** (`heart-disease-predictor/`).

**Linux/Mac:**

```bash
make setup     # creates .venv and installs requirements.lock
make notebook  # opens JupyterLab, train the model here first
make api       # starts the FastAPI server at localhost:8000
make app       # starts the Streamlit interface at localhost:8501
make docker    # builds and runs the Docker image
make clean     # removes the .venv
```

**Windows (PowerShell):**

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.lock
```

To regenerate the lock file after updating `pyproject.toml`:

```bash
pip install pip-tools
pip-compile pyproject.toml --extra dev --output-file requirements.lock
```

## How to run

> All commands below must be executed from the project root.

> **Note:** the `models/` folder is not versioned. You must run step 1 before using the API or the Streamlit interface.

**1. Train the model**

Open and run `notebooks/heart_disease.ipynb`.  
This generates `models/mlp_heart.pt` and `models/scaler.pkl`.

```bash
make notebook
# or on Windows:
.venv\Scripts\jupyter lab notebooks\
```

**2. Streamlit interface**

```bash
make app
# or on Windows:
.venv\Scripts\streamlit run app/app.py
```

Opens at `http://localhost:8501`, patient data form with probability result and visual gauge.

**3. FastAPI**

```bash
make api
# or on Windows:
.venv\Scripts\uvicorn api.main:app --reload --host 0.0.0.0 --port 8000
```

Interactive documentation available at `http://localhost:8000/docs`.

**4. Request example**

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "age": 52, "sex": 1, "cp": 0, "trestbps": 125,
    "chol": 212, "fbs": 0, "restecg": 1, "thalach": 168,
    "exang": 0, "oldpeak": 1.0, "slope": 2, "ca": 2, "thal": 7
  }'
```

Expected response:

```json
{
  "disease_probability": 0.7823,
  "diagnosis": "Disease",
  "threshold_used": 0.5
}
```

**5. Via Docker**

```bash
make docker
# or on Windows (requires Docker Desktop):
docker build -t heart-predictor .
docker run -p 8000:8000 heart-predictor
```

## Technical details

**Architecture:** MLP with two hidden layers (64 and 32 neurons), Batch Normalization and Dropout for regularization.

**Training:** Adam with L2 weight decay, early stopping and automatic learning rate reduction. Evaluated with AUC-ROC, F1 and threshold analysis to minimize false negatives.

**Reproducibility:** isolated environment via `.venv`, pinned dependencies in `requirements.lock`, fixed SEED.
